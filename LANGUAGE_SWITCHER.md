## 🎯 工作原理

### 语言优先级
1. **用户选择的语言** - 存储在 `chrome.storage.sync.language`
2. **浏览器语言** - 作为默认值（中文浏览器 → zh_CN，其他 → en）

### 切换流程
1. 用户点击语言切换按钮 🌐
2. 显示语言选择菜单（中文/English）
3. 用户选择语言
4. 保存到 `chrome.storage.sync`
5. 显示切换成功提示
6. 提示用户重新打开弹窗

### 为什么需要重新打开？
- Chrome 扩展的 `chrome.i18n` API 在页面加载时确定语言
- 无法在运行时动态切换
- 重新打开弹窗会重新加载，应用新语言

## 🔧 技术实现细节

### i18n.js 增强
- 添加 `getCurrentLocale()` - 获取当前语言（用户选择 > 浏览器语言）
- 添加 `loadTranslations()` - 动态加载翻译文件
- 优先使用 `chrome.i18n` API，失败时使用加载的翻译数据
- 支持占位符替换（$1, $2...）

### popup.js 新增函数
- `toggleLanguageDropdown()` - 显示/隐藏语言菜单
- `switchLanguage(language)` - 切换语言并保存
- 修改事件监听器，支持语言选项点击
