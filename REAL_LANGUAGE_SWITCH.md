## 🎯 工作原理

### 新的 i18n.js
- 完全独立的翻译系统
- 从 `_locales/` 目录加载 JSON 文件
- 支持运行时动态切换
- 支持占位符替换

### 语言切换流程
1. 点击 🌐 按钮
2. 选择语言（中文/English）
3. **立即切换** - 所有文本实时更新
4. 保存到 `chrome.storage.sync`
5. 下次打开自动应用

### 初始化流程
1. 检查用户是否保存了语言偏好
2. 如果有 → 使用用户偏好
3. 如果没有 → 检测浏览器语言（zh → 中文，其他 → 英文）
4. 加载对应的翻译文件
5. 应用到页面

## 🔧 技术细节

### 与之前的区别

#### 之前的实现（失败）
```javascript
// 使用 chrome.i18n.getMessage()
const text = chrome.i18n.getMessage(key);
// ❌ 无法运行时切换
```

#### 现在的实现（成功）
```javascript
// 完全自定义的 i18n 系统
const text = i18n(key);  // 从加载的 JSON 读取
await switchLanguage('en');  // 重新加载并应用
// ✅ 立即切换
```

### 关键函数

#### `initLanguage()`
- 检查用户偏好
- 检测浏览器语言
- 加载翻译文件

#### `switchLanguage(lang)`
- 加载新语言的翻译
- 更新全局变量
- 保存到存储
- 重新应用翻译（调用 `applyTranslations()`）

#### `applyTranslations()`
- 遍历所有 `[data-i18n]` 元素
- 遍历所有 `[data-i18n-title]` 元素
- 遍历所有 `[data-i18n-placeholder]` 元素
- 更新文本内容

## ⚠️ 已知限制

### manifest.json 的名称和描述
- 扩展名称和描述仍然使用 `chrome.i18n`
- 这部分**无法**在运行时切换
- 跟随浏览器语言
- 这是 Chrome 扩展的限制，无法绕过

### 解决方案
- 使用通用名称："PRISM Lens / PRISM 棱镜"
- 描述同时包含中英文
- 或者使用纯英文（国际化通用）
